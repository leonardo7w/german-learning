Adjektivendungen

Wann heißt es der gute Code, ein guter Code oder einfach guter Code? Die eine Regel, die alles entscheidet.

Dein Ziel. In Lektion 01 hast du die vier Fälle gelernt. Bei den Adjektivendungen werden die Fälle ernst: Jedes Mal, wenn du etwas beschreibst — ein erfahrener Architekt, mit agilen Methoden —, musst du die Endung sofort wählen. Genau hier wackeln die meisten auf der Mittelstufe. Beherrsche die eine Regel unten, und du rätst nicht mehr. Etwa 25 Minuten; die Übungen steigern sich bis B2.

iDie eine Regel

Das Genus-/Kasus-Signal (die Markierung -r / -e / -s / -m / -n)
muss genau einmal in der Wortgruppe stehen —
am Artikel oder am Adjektiv. Nie beides, nie keines.

Dieses eine Prinzip erzeugt alle drei „Deklinationen". Frag dich: Zeigt der Artikel Kasus und Genus schon klar?

SCHWACH Der Artikel macht die Arbeit (nach der/die/das, dieser, jeder, welche, alle…). Das Adjektiv entspannt sich → nur -e oder -en.
der gute Code · dem guten Code

GEMISCHT Der Artikel ist halb blind (nach ein/kein/mein, dein, sein, ihr, unser…). „ein" zeigt nichts für mask.-Nom. und Neutrum; dort füllt das Adjektiv die Lücke mit der starken Endung, sonst schwach.
ein guter Code · ein gutes Team · einen guten Code

STARK Gar kein Artikel (oder nach etwas, viel, wenig, Zahlen). Das Adjektiv trägt das volle Signal — es übernimmt die der/die/das-Endungen.
guter Code · kaltes Wasser · mit gutem Code

iiDie drei Tabellen

Gelb = die Felder, die man sich merken sollte; der Rest folgt dem Muster.

SCHWACH nach der-Wörtern — nur -e / -en

KasusmaskfemneutrPlural
Nom-e-e-e-en
Akk-en-e-e-en
Dat-en-en-en-en
Gen-en-en-en-en

Kurz: überall -en, außer den 5 gelben -e-Feldern oben links.

GEMISCHT nach ein-Wörtern — schwach, mit 3 Korrekturen

KasusmaskfemneutrPlural
Nom-er-e-es-en
Akk-en-e-es-en
Dat-en-en-en-en
Gen-en-en-en-en

Wie schwach, außer 3 gelben Feldern, wo „ein" nichts zeigt → das Adjektiv nimmt die starke Endung (-er mask.-Nom, -es neutr. Nom/Akk).

STARK ohne Artikel — Adjektiv = das der/die/das-Signal

KasusmaskfemneutrPlural
Nom-er-e-es-e
Akk-en-e-es-e
Dat-em-er-em-en
Gen-en-er-en-er

Spiegelt der/den/dem/des… (der→-er, das→-es, dem→-em). Einzige Besonderheit: Gen mask./neutr. ist -en (das -s des Substantivs markiert schon): trotz schlechten Wetters.

Druckbare Fassung: Spickzettel zu den Adjektivendungen. Lass auch den Kasus-Spickzettel offen — ohne den Kasus kannst du keine Endung wählen.

iiiDie 4-Schritte-Methode

Lauf bei jedem Adjektiv das im Kopf durch:

1. Welcher Artikel? der-Wort → SCHWACH · ein-Wort → GEMISCHT · keiner → STARK.
2. Welcher Kasus? (Subjekt? Objekt? nach einer Präposition? → siehe Lektion 01.)
3. Welches Genus/Numerus hat das Substantiv?
4. Lies die Endung aus der richtigen Tabelle ab.

AWelches Muster?

Benenne nur die Deklination — die genaue Endung ist noch egal.

BSCHWACH  Ergänze die Endung (nach der-Wörtern)

CGEMISCHT  Ergänze die Endung (nach ein-Wörtern)

DSTARK  Ergänze die Endung (ohne Artikel)

ESchwere Stufe: die Präposition bestimmt den Kasus

Jetzt gibt dir nicht der Artikel den Kasus, sondern eine Präposition. Denk an Lektion 01: mit/bei/aus/zu → Dat; für/durch/ohne → Akk; in/auf/an → Dat (Ort) oder Akk (Richtung). Dann wähl die Endung. B1–B2

FFinde den Fehler

Richtig oder falsch? Achte auf die Fallen „Signal doppelt" und „Signal fehlt".

GSzene: Einstellung in deinem Zürcher Team

Ergänze jede Endung in diesem Dialog aus dem Software-Team. Mehrere Adjektive, echter Kasus-Druck. B2

HKarteikarten — die Felder, die stolpern lassen

Das sind die Felder, die wirklich Fehler verursachen. Dreh sie um, bis es sitzt.

Ich bin dein Lehrer — fordere mich

Das ist ein schweres Thema; Stolpern ist normal. Schreib mir im Chat:

  • „Ich verwechsle gemischt und stark — drill nur die beiden."
  • „Gib mir 10 Sätze aus der Softwarearchitektur zum Übersetzen."
  • „Frag mich mündlich ab, ohne Multiple Choice."
  • „Mein Ergebnis war ___ / ___" — damit ich die nächste Lektion abstimme.
